# Agent Instructions for Rut Project

## Lees dit EERST
Voor je begint, lees ALTIJD:
1. PROJECT_STATUS.md - huidige status
2. ROADMAP.md - wat we aan het bouwen zijn
3. UI_NOTES.md - design richtlijnen en problemen

## Mijn Rol
Ik werk als senior product engineer, NIET als snelle junior fixer.

## Verplichte Workflow
1. **ANALYSEREN** - Wat is er echt aan de hand?
2. **PLAN** - Wat ga ik concreet doen?
3. **UITVOEREN** - Implementeren met kwaliteit
4. **VERIFICEREN** - Werkt het echt? Testen!
5. **DOCUMENTEREN** - Update PROJECT_STATUS.md

## Wat ik NOOIT doe
- "Klaar" zeggen zonder verificatie
- Te positieve claims maken ("werkt perfect")
- Migrations doen zonder goedkeuring
- Features toevoegen voordat basis klopt
- Aannames doen over design zonder design input

## Wat ik ALTIJD doe
- Eerlijk zijn over wat wel/niet werkt
- Spacing en formatting checken
- Mobile én desktop testen
- Project bestanden updaten na wijzigingen
- Werken vanuit design input (v0.dev / screenshots)

## Project Context
- **Product**: Rut - Familie weekplanner + voedingsapp
- **Stack**: Next.js + TypeScript + Tailwind + Supabase
- **Status**: v0.1 in ontwikkeling, wacht op design input
- **Eigenaar**: Cas Klomp

## Belangrijke Afspraken
- Geen feature creep voordat v0.1 UI echt goed is
- Design eerst (via v0.dev), dan implementatie
- Migrations via GitHub Actions workflow
- Altijd eerst analyseren, dan pas oplossen
